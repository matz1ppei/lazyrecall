# Backlog

## 機能追加

### 学習・復習
- [ ] **DBファイルの場所を固定する** — 現状はカレントディレクトリに `lazyrecall.db` が生成される。`~/.local/share/lazyrecall/lazyrecall.db` などに固定しないと、別ディレクトリから起動するとデータが消えたように見える
- [ ] **1日の上限をユーザーが設定できるようにする** — 現状は `dailyReviewLimit = 100` にハードコード。`.env` か設定画面で変更できると良い
- [ ] **復習中に今日の進捗を表示する** — セッション中は「3 / 20」（セッション内）しか見えない。「43 / 100 今日」も表示するとモチベーションになる

### Match Madness
- [ ] **Streak 表示** — 連続正解数をゲーム中に表示する
- [ ] **難易度選択** — ペア数を 4 / 6 / 8 から選べるようにする

### 多言語対応
- [ ] **英語学習時のBack言語を日本語にする** — 現状はどの言語でも Back（裏面）が英語固定。英語を学習対象にした場合は Back を日本語にする必要がある。`GenerateCardsForWords` のプロンプトを母語設定に応じて切り替える、または言語ごとに Back 言語を設定できるようにする

### Blank Fill
- [ ] **活用形対応（example_word フィールド）** — 現状、AI が例文で活用形（hablo など）を使うと base form（hablar）とマッチせずブランクが作れない。暫定対応として該当カードを除外しているが、根本解決としてカードに `example_word TEXT` カラムを追加し、AI生成時にそのカードの例文で実際に使った語形を保存して置換に使う。`GenerateCardsForWords` プロンプトに `"example_word": "<form used in sentence>"` を追加するだけで対応可能。

### コンテンツ生成
- [ ] **画像生成** — 例文から Stable Diffusion 等で画像生成。記憶と画像の結びつきを活用（`draft.md` より）
- [ ] **Claudeモデルを設定可能に** — 現状 `claude-haiku-4-5` にハードコード。環境変数で切り替えられると良い

### 除外リスト
- [x] **単語除外リスト** — 頻出すぎて学習不要な単語（英語の "this" / "I" / "the" など）を登録しておき、auto-add・Fetch: Language・セットアップの候補から除外する。除外リストの管理方法は以下を検討:
  - `~/.config/lazyrecall/exclude.txt`（1行1単語、`#` でコメント）が最もシンプル
  - 設定画面（`[c]`）に "Edit exclusions" 項目を追加してTUI内で編集できると便利
  - 既存カードと同様に `db.GetAllFronts()` による重複除外は維持しつつ、除外リストは「そもそも候補に出さない」前段フィルターとして機能させる

### データ管理
- [ ] **バックアップ機能** — `lazyrecall.db` が1ファイルなので削除したら終わり。`~/.local/share/lazyrecall/backups/` への自動バックアップ
- [ ] **エクスポート機能** — CSV エクスポート（インポートは実装済み）
